# 概念

简单来说，**批处理 (Batch Processing)** 就是将一批 SQL 语句先“攒”起来，然后通过一次网络往返（Network Round-trip）发送给数据库服务器执行，而不是来一条发一条。

- **普通模式**：Java 发送 1 条 -> 数据库执行 1 条 -> 数据库返回结果 -> Java 发送下 1 条...（就像送外卖，一份一份送）。
    
- **批处理模式**：Java 攒够 100 条 -> 一次性发给数据库 -> 数据库统一执行 -> 返回结果（就像快递大货车，一车送 100 件）。

## 代码

~~~java
try (Statement stmt = conn.createStatement()) {
    // 1. 关闭自动提交，这对性能至关重要
    conn.setAutoCommit(false);

    // 2. 将多条完整的 SQL 字符串加入批处理
    stmt.addBatch("insert into user (name, age) values ('小刚', 18)");
    stmt.addBatch("insert into user (name, age) values ('小强', 18)");
    stmt.addBatch("update user set age = 20 where name = '小王'");
    stmt.addBatch("delete from user where name = '小黑子'");

    // 3. 一次性发送到数据库执行
    int[] results = stmt.executeBatch();
    这个数组里存放的是每一条 SQL 语句执行后的结果（即受影响的行数）

    // 4. 提交事务
    conn.commit();
} catch (SQLException e) {
    conn.rollback(); // 如果出错了，全部回滚
    e.printStackTrace();
}
~~~
## 底层是怎么运作的？

我们可以从三个层面简单拆解它的“黑盒”：
### 客户端缓冲区 (Java 端)

当你调用 `addBatch()` 时，JDBC 驱动程序并没有联系数据库，而是在本地内存中维护了一个**列表（List 或 Buffer）**。它只是把你的参数和 SQL 结构暂存起来。

### 网络传输

当你调用 `executeBatch()` 时，驱动程序会将这个列表打包成一个特定的数据包，通过 Socket 发送给数据库。

- **如果不刷批**：1000 条 SQL 意味着 1000 次网络请求 + 1000 次响应。
    
- **如果刷批**：1000 条 SQL 压缩成 1 次请求 + 1 次响应。网络开销（Latency）骤降。
    

### 数据库端处理

数据库接收到这一大包数据后，会进行优化处理。

- **预编译重用**：数据库发现这些 SQL 的结构都一样（只是参数不同），它只需解析（Parse）一次 SQL 结构，然后把不同的参数填进去快速运行。
    
- **驱动优化（MySQL 特有）**： 如果你在数据库连接字符串中加入了 `rewriteBatchedStatements=true`，MySQL 驱动会把多条 `INSERT` 语句合并成一条极其高效的长语句： `INSERT INTO user VALUES (1, 'A'), (2, 'B'), (3, 'C')...;`
