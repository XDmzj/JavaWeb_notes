


# 题目
在一张daily_profits表中，存储了公司每天的利润记录。请计算每一天的累计利润，并按profit_date升序输出所有字段。

具体要求如下：

1. 计算每一天的累计利润。
2. 输出结果按profit_date升序排列。


### daily_profits表

|profit_id|profit_date|profit|
|---|---|---|
|1|2024-01-01|100.00|
|2|2024-01-02|150.00|
|3|2024-01-03|200.00|
|4|2024-01-04|250.00|
|5|2024-01-05|300.00|

### 输出示例表格

|profit_id|profit_date|profit|cumulative_profit|
|---|---|---|---|
|1|2024-01-01|100.00|100.00|
|2|2024-01-02|150.00|250.00|
|3|2024-01-03|200.00|450.00|
|4|2024-01-04|250.00|700.00|
|5|2024-01-05|300.00|1000.00|


# 题解
```sql
select profit_id,profit_date,profit,
sum(profit)
over(
    order by profit_date
)
as
cumulative_profit
from daily_profits
order by profit_date
;
```

# 窗口函数
## 1. 什么是窗口函数？

窗口函数允许我们在不通过 `GROUP BY` 压缩行数的情况下，执行跨行的聚合计算。

- **核心逻辑**：它为每一行数据开辟一个“窗口”，在窗口内计算结果，并将结果贴在当前行的后面。
    
- **结果行数**：输入 100 行，输出依然是 100 行。

---

## 2. 基本语法结构

```
<函数名>(<列名>) OVER (
    PARTITION BY <分组列>
    ORDER BY <排序列>
    ROWS/RANGE <窗口框架>
)
```

### 参数详解：

1. **PARTITION BY**：逻辑分堆。计算在每个分区内部独立进行，跨分区则重新计数。像group by
    
2. **ORDER BY**：定义计算顺序。对于排名函数必选，对于聚合函数会触发“累计计算”。
    
3. **ROWS/RANGE**：精细控制窗口边界（例如：计算“前三行到当前行”）。
    

---

## 3. SUM 函数的两种面孔

`SUM()` 在不同子句下表现完全不同：

|**特性**|**搭配 GROUP BY**|**搭配 OVER()**|
|---|---|---|
|**数据形态**|“塌陷”，多行变一行|保留，原样输出每一行|
|**显示明细**|无法显示（除非在分组中）|可以同时显示明细和聚合值|
|**混合使用**|不能直接与非分组字段混合|可以直接在 `SELECT` 中展示|

---

## 4. 深度解析：为什么 `ORDER BY` 会导致累计求和？

当你写 `sum(salary) OVER(ORDER BY id)` 时，SQL 默认补全了窗口框架参数：

`RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW`

- **执行逻辑**：
    
    1. 按 `id` 排序。
        
    2. 窗口范围锁定为：**从第一行到当前行**。
        
    3. 每一行都在前面所有行的基础上进行加和，形成“滚雪球”效应。
        

---

## 5. 常见参数组合逻辑表

|**OVER() 内部写法**|**计算范围**|**结果表现**|
|---|---|---|
|**`( )`**|全表|每一行都显示全表总和。|
|**`(PARTITION BY dept)`**|部门内|每一行显示自己部门的总和。|
|**`(ORDER BY date)`**|全表累计|显示全表从开始到当前的累计值。|
|**`(PARTITION BY dept ORDER BY date)`**|部门累计|显示部门内部随时间推移的累计值。|

---

## 6. SQL 执行顺序中的窗口函数

窗口函数在 SQL 执行链条中处于较后的位置：

1. `FROM` & `JOIN`
    
2. `WHERE`
    
3. `GROUP BY`
    
4. `HAVING`
    
5. **窗口函数 (`SELECT` 阶段)**
    
6. `ORDER BY` (全表排序)
    
7. `LIMIT`
    

> **面试避坑点**：因为窗口函数在 `WHERE` 之后执行，所以你不能直接在 `WHERE` 里写 `rank = 1`。必须使用子查询或 CTE（公用表表达式）。
