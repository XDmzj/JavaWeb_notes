## 要求
请你写一个SQL查询，计算不同学校、不同难度的用户平均答题量，根据示例，你的查询应返回以下结果(结果在小数点位数保留4位，4位之后四舍五入)：

|   |   |   |
|---|---|---|
|university|difficult_level|avg_answer_cnt|
|北京大学|hard|1.0000|
|复旦大学|easy|1.0000|
|复旦大学|medium|1.0000|
|山东大学|easy|4.5000|
|山东大学|medium|3.0000|
|浙江大学|easy|5.0000|
|浙江大学|medium|2.0000|


# 题解

~~~sql
select university , difficult_level ,
round(count(*) / count(distinct up.device_id) , 4) as avg_answer_cnt
from user_profile up
inner join question_practice_detail qpd on up.device_id = qpd.device_id
inner join question_detail qd ON qpd.question_id = qd.question_id
group by up.university , qd.difficult_level
order by up.university, qd.difficult_level;
~~~



## 涉及知识点
### 1. `ROUND` 函数：数值修饰符

`ROUND` 用于对数字进行**四舍五入**。

- **语法**：`ROUND(数值, 保留位数)`
    
- **使用说明**：
    
    - 如果保留位数为正数，则对小数点后进行四舍五入。
        
    - 如果保留位数为 0，则取整。
        
    - **示例**：`ROUND(3.14159, 2)` 结果为 `3.14`。在你的代码中，`ROUND(..., 4)` 确保了平均值不会出现冗长的长尾小数。

---

### 2. `COUNT` 函数：计数器

`COUNT` 用于统计行数或非空值的数量。

- **语法**：`COUNT(列名)` 或 `COUNT(*)`
    
- **关键区别**：
    
    - `COUNT(*)`：统计结果集中的**总行数**（包括 NULL 值）。
        
    - `COUNT(列名)`：只统计该列中**非 NULL** 的值。
        
    - `COUNT(DISTINCT 列名)`：统计该列中**去重后**的唯一值数量。
        
- **代码应用**：`COUNT(qpd.id)` 算的是总答题次数，而 `COUNT(DISTINCT u.device_id)` 算的是有多少个独立的用户参与了答题。

---

### 3. 三个表可以直接连接吗？

**可以，但必须有“纽带”。**

在 SQL 中，连接表的数量没有理论上限，但直接连接的前提是它们之间存在**关联字段**（通常是外键）。

- **连接路径**：你的代码通过 `qpd`（答题记录表）作为桥梁：
    
    1. `user_profile` 与 `qpd` 通过 `device_id` 连接。
        
    2. `qpd` 与 `question_detail` 通过 `question_id` 连接。
        
- **注意**：如果没有关联字段直接写 `FROM A, B`，会产生**笛卡尔积**（A表的每一行和B表的每一行强行组合），导致数据爆炸且毫无意义。
    

---

### 4. JOIN 的区别与联系

在 SQL 中，`JOIN` 通常是连接方式的统称，但在大多数数据库（如 MySQL, PostgreSQL）中，**单独写 `JOIN` 默认就是 `INNER JOIN`**。

 **可视化对比：**

|**类型**|**描述**|**结果**|
|---|---|---|
|**INNER JOIN** (内连接)|只保留两个表中**完全匹配**的行。|如果一个学生没做过题，或者一道题没被任何人做过，这些数据都会消失。|
|**LEFT JOIN** (左外连接)|保留**左表**所有行，右表匹配不上的填 **NULL**。|如果你想看“所有学生”的答题情况，哪怕他一次没做过（显示为0），就得用 `user_profile LEFT JOIN ...`。|
|**RIGHT JOIN** (右外连接)|保留**右表**所有行，左表匹配不上的填 NULL。|与 LEFT JOIN 逻辑相反，较少使用，通常可以互相转化。|
