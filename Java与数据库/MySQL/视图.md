# 语法

## 创建视图 (CREATE VIEW)

创建视图的基本逻辑是将一条 `SELECT` 语句存储起来，并赋予其一个名称。

```SQL
CREATE VIEW 视图名称 [(列名1, 列名2, ...)]
AS
SELECT_语句
[WITH [CASCADED | LOCAL] CHECK OPTION];
```

- **列名列表**: 可选，如果在 `SELECT` 中使用了别名或聚合函数，建议在这里明确指定视图的列名。
    
- **WITH CHECK OPTION**: 可选，用于约束通过视图进行的 `INSERT/UPDATE` 操作必须满足视图定义的条件（过滤条件）。


### WITH CHECK OPTION

当视图存在层级关系（即视图 A 基于视图 B 创建）时，`CHECK OPTION` 的检查范围取决于你选用的模式。

#### `WITH CASCADED CHECK OPTION`（最常用/默认）

- **逻辑**：严格检查。无论是在当前视图还是底层引用的视图，只要定义了过滤条件，所有条件都必须满足。
    
- **行为**：插入或更新时，数据库会向上追溯，检查该表所有相关的视图定义，任何一个环节违规，直接报错。
    

####  `WITH LOCAL CHECK OPTION`

- **逻辑**：仅检查当前视图。
    
- **行为**：仅检查当前视图的 `WHERE` 子句。如果当前视图引用了其他视图，它不会强制去检查底层视图的条件（除非底层视图自己也加了 `CHECK OPTION`）。
    

---

#### 直观示例

假设我们要维护一个“高分用户”列表（`score > 80`）：

```
-- 1. 创建视图
CREATE OR REPLACE VIEW view_high_score_users AS
SELECT * FROM users WHERE score > 80
WITH CHECK OPTION; -- 默认为 CASCADED

-- 2. 尝试更新
-- 如果执行：UPDATE view_high_score_users SET score = 50 WHERE id = 101;
-- 结果：数据库直接拒绝！因为 score = 50 不满足视图定义的 score > 80。
```

## 修改视图 (ALTER VIEW)

### `CREATE OR REPLACE VIEW`

这是目前主流数据库（如 MySQL、PostgreSQL、Oracle）推荐的做法。它无需先删除视图，直接执行该语句即可覆盖旧的视图定义，非常高效。

**语法结构：**

```
CREATE OR REPLACE VIEW 视图名称 AS
SELECT 字段1, 字段2, ...
FROM 表名
WHERE 条件;
```

- **优点**：原子性操作，不容易因为忘记删除旧视图而导致错误，且保留了视图的权限（Permissions）设置（取决于具体数据库实现）。

---

### 先 `DROP` 再 `CREATE`

对于一些不支持 `OR REPLACE` 语法的数据库系统，或者当你需要彻底重置视图逻辑时，这是最稳妥的方案。

**语法结构：**

```
-- 1. 删除旧视图
DROP VIEW IF EXISTS 视图名称;

-- 2. 创建新视图
CREATE VIEW 视图名称 AS
SELECT 字段1, 字段2, ...
FROM 表名
WHERE 条件;
```

- **注意**：如果你在生产环境操作，且该视图被多个其他对象（如存储过程、其他视图）所依赖，使用 `DROP` 时需谨慎，建议先确认依赖关系。
    

---

### ALTER VIEW

部分数据库（如 SQL Server）支持直接使用 `ALTER` 关键字。

**语法结构（SQL Server 为例）：**

```
ALTER VIEW 视图名称 AS
SELECT 字段1, 字段2, ...
FROM 表名
WHERE 条件;
```

- **特点**：直接修改视图定义，而不影响视图的权限，即该视图原本被授予给其他用户的访问权限不会丢失。

### UPDATE
当你在视图上执行 `UPDATE` 时，它其实是在做两件事：

1. **映射到底层表**：视图本身没数据，它会把你的 `UPDATE` 请求“翻译”给原始表。
    
2. **触发 `CHECK OPTION` 校验**（如果你定义了它）：这是为了防止你把数据改到视图“看不见”的地方。


---

## 删除视图 (DROP VIEW)

删除视图不会影响底层物理表（Base Table）中的数据，只会移除视图定义。

```
DROP VIEW [IF EXISTS] 视图名称 [RESTRICT | CASCADE];
```

- **IF EXISTS**: 防止视图不存在时报错。
    
- **RESTRICT / CASCADE**:
    
    - `RESTRICT`: 如果有其他对象（如存储过程或另一个视图）依赖该视图，则删除失败（这是默认值）。
        
    - `CASCADE`: 级联删除，将所有依赖该视图的对象一并删除。
