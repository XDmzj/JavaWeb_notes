# 基础语法


触发器就像其名字一样，在某种条件下会自动触发，在`select`/`update`/`delete`时，会自动执行我们预先设定的内容，触发器通常用于检查内容的安全性，相比直接添加约束，触发器显得更加灵活。

触发器所依附的表称为基本表，当触发器表上发生`select`/`update`/`delete`等操作时，会自动生成两个临时的表（new表和old表，只能由触发器使用）

比如在`insert`操作时，新的内容会被插入到new表中；在`delete`操作时，旧的内容会被移到old表中，我们仍可在old表中拿到被删除的数据；在`update`操作时，旧的内容会被移到old表中，新的内容会出现在new表中。

```
-- 1. 改变定界符为 $$
DELIMITER $$

-- 2. 开始编写包含多个分号的复杂触发器
CREATE TRIGGER cfq_example
AFTER UPDATE ON employees
FOR EACH ROW
BEGIN
    -- 这里的 ; 是代码的一部分，不会被 MySQL 提前执行
    INSERT INTO audit_log (msg) VALUES ('update');
    UPDATE logs SET status = 1 WHERE id = 1;
END$$  -- 触发器完整结束点

-- 3. 改回默认的 ;
DELIMITER ;
```

FOR EACH ROW表示针对每一行都会生效，无论哪行进行指定操作都会执行触发器！

## DELIMITER作用

什么是 `DELIMITER`？

`DELIMITER` 的字面意思是定界符”或“分隔符”。在数据库（特别是 MySQL）中，它是一个用来**改变 SQL 语句结束标志**的命令。

为什么需要它？（核心逻辑）

在默认情况下，MySQL 使用分号 `;` 作为 SQL 语句的结束标志。当你输入 `SELECT * FROM users;`，数据库遇到分号，就知道这条语句写完了，立即执行。

但是，当你编写**存储过程、触发器或函数**时，代码块内部会有多条 SQL 语句，每条语句后面都会有分号 `;`。

- **如果不改定界符**：数据库在读取你的触发器代码时，读到第一条 SQL 的分号，就以为触发器写完了，剩下的代码会因为它不完整而报错。
    
- **使用 `DELIMITER` 的作用**：你先告诉数据库：“从现在起，不要把 `;` 当作结束标志，改用 `$$`（或其他字符）作为结束标志。” 这样你就可以在代码块里随意写 `;` 而不会被打断，直到你遇到那个自定义的 `$$`。



# 查看触发器

通过下面的命令来查看触发器：

```sql
SHOW TRIGGERS
```

# 删除触发器

如果不需要，我们就可以删除此触发器：

```sql
DROP TRIGGER 触发器名称
```
