在 MyBatis 中，**接口绑定（Interface Binding）** 是一个让 Java 代码和 XML 映射文件“握手”的核心机制。

简单来说，它让你能够像调用普通 Java 方法一样执行 SQL 语句，而不需要再手动去调用 `session.selectList("statementId")` 这种容易写错字符串的原始方式。

---

### 1. 它是如何工作的？

在没有接口绑定时，你需要记住 XML 里的 `namespace` 和 `id`，然后手动调用。有了接口绑定后，你只需要定义一个 **Mapper 接口**。

**核心要求：**

1. **接口的全类名** 必须等于 XML 中的 `namespace`。
    
2. **接口的方法名** 必须等于 XML 中 SQL 语句的 `id`。
    
3. **方法的参数和返回值** 必须与 XML 中定义的 `parameterType` 和 `resultType` 匹配。
    

---

### 2. 代码示例对比

#### ❌ 原始方式（硬编码字符串）：

这种方式容易写错 `student.selectAll` 这个字符串，且没有类型检查。

Java

```java
List<Student> list = session.selectList("student.selectAll");
```

#### ✅ 接口绑定方式：

**第一步：定义接口**

```java
package com.example.mapper;
public interface StudentMapper {
    List<Student> selectAll(); // 方法名要对应 XML 的 id
}
```

**第二步：配置 XML**

```xml
<mapper namespace="com.example.mapper.StudentMapper">
    <select id="selectAll" resultType="Student">
        select * from student
    </select>
</mapper>
```

**第三步：使用**

```java
// MyBatis 会利用动态代理自动帮你生成这个接口的实现类
StudentMapper mapper = session.getMapper(StudentMapper.class);
List<Student> list = mapper.selectAll();
```

---

### 3. 为什么要用接口绑定？（好处）

- **类型安全**：编译器会帮你检查参数类型和返回值，减少运行时报错。
    
- **IDE 支持**：你可以直接通过 `Ctrl + 左键` 在 Java 方法和 XML SQL 之间跳转（如果你安装了 MyBatis 插件）。
    
- **解耦**：业务逻辑层（Service）只依赖接口，不需要关心底层的 SQL 是怎么实现的。
    
- **动态代理**：MyBatis 利用 JDK 动态代理技术，在运行时自动为你创建接口的实现类，你不再需要写 `MapperImpl`。
    

---

### 4. 接口绑定的实现方式

除了使用 XML，接口绑定还有一种更直接的方式：**注解**。

```java
public interface StudentMapper {
    @Select("select * from student")
    List<Student> selectAll();
}
```

- **小型项目**：可以用注解，简单快捷。
    
- **大型项目（如你的实验平台）**：建议使用 **XML 绑定**。因为 SQL 往往很复杂，写在 XML 里更易于维护和调试，也能利用刚才聊到的 `static` 块或 `resultMap` 等高级特性。
