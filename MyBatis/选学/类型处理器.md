
# 概念
在我们前面的学习中，相信各位小伙伴应该有留意到一个叫做`typeHandler`的属性，在正常情况下，我们使用Mybatis得到结果都能够自动完成属性类型的的推断，比如：

```java
public class User {
    int id;
    String name;
    int age;
}
```

这里类中的所有字段类型实际上在查询时都可以被正确进行解析，这是因为Mybatis默认情况下对这些JDK提供的类型做了支持，因此他们可以直接正确进行解析，详细列表：[https://mybatis.org/mybatis-3/zh_CN/configuration.html#typeHandlers](https://mybatis.org/mybatis-3/zh_CN/configuration.html#typeHandlers)

但是有些时候可能我们的对象字段并不是JDK提供的类型，尤其是后面各位小伙伴在后续Spring阶段的学习中接触到JSON这种数据时，数据的转换就会变得很重要，这里我们通过一个简单例子来进行讲解：

```java
@Data
public class MyString {   //现在我们自己写了一个对String包装类型
    private final String text;
}
```

接着我们将实体类的对应的字段进行修改：

```java
public class User {
    int id;
    MyString name;   //使用我们自己定义的String包装类
    int age;
}
```

此时，我们希望的是Mybatis可以正确将数据库中的name字段读取出来，并自动转化为一个MyString对象，里面的text值为查询name得到的结果，但这毕竟是理想情况，实际上Mybatis压根不知道这种类型该怎么进行数据的映射，就直接报错了：

```
### Error building SqlSession.
### The error may exist in com/test/mapper/TestMapper.java (best guess)
### Cause: org.apache.ibatis.builder.BuilderException: Error parsing SQL Mapper Configuration. Cause: java.lang.IllegalStateException: Type handler was null on parameter mapping for property 'name'. It was either not specified and/or could not be found for the javaType (com.test.handler.MyString) : jdbcType (null) combination.
	at org.apache.ibatis.exceptions.ExceptionFactory.wrapException(ExceptionFactory.java:30)
```
# TypeHandler手动实现
这里Mybatis提示我们找不到这个类型的Type handler，也就是Mybatis并不知道这种类型的数据该怎么进行映射，映射规则是什么。此时我们就可以手动提供一个TypeHandler实现，要编写自定义的类型处理器，我们需要：

```java
public class MyTypeHandler extends BaseTypeHandler<MyString> {
    @Override
    public void setNonNullParameter(PreparedStatement ps, int i, MyString parameter, JdbcType jdbcType) throws SQLException {
        
    }

    @Override
    public MyString getNullableResult(ResultSet rs, String columnName) throws SQLException {
        return null;
    }

    @Override
    public MyString getNullableResult(ResultSet rs, int columnIndex) throws SQLException {
        return null;
    }

    @Override
    public MyString getNullableResult(CallableStatement cs, int columnIndex) throws SQLException {
        return null;
    }
}
```

可以看到这里我们需要实现很多种不同的方法，我们一个一个来介绍。

首先我们需要在类的最上方添加一个`@MappedJdbcTypes`注解，用于指定当前类型处理器是用于处理对应的数据库什么类型的数据的：

```java
@MappedJdbcTypes(JdbcType.VARCHAR)  //这里我们映射的是数据库里面的varchar
public class MyTypeHandler extends BaseTypeHandler<MyString> {
  ...
}
```


## 将我们的Java类型转换为对应的JDBC类型

接着是四个方法的实现，它们涉及到两种操作，一个是如何将我们的Java类型转换为对应的JDBC类型，还有一个就是如何将JDBC类型转换为对应的Java类型。

首先是如何将Java转JDBC类型，这里的PreparedStatement我们在JDBC阶段已经讲过，不再赘述：

```java
@Override
public void setNonNullParameter(PreparedStatement ps, int i, MyString parameter, JdbcType jdbcType) throws SQLException {
  	//这里的i就是PreparedStatement的第几个参数，直接使用setString设置即可
    ps.setString(i, parameter.getText());  //按照我们自己的方式把需要插入的值设置进去
}
```


## 还有一个就是如何将JDBC类型转换为对应的Java类型
剩下三个都是JDBC类型和Java类型的转换，只是参数不同而已，我们就简单编写一下：

```java
//用于配置如何从ResultSet中读取并转换为我们需要的类型
@Override
public MyString getNullableResult(ResultSet rs, String columnName) throws SQLException {
    String value = rs.getString(columnName);  //这里提供的是columnName进行查询
    return new MyString(value);
}

@Override
public MyString getNullableResult(ResultSet rs, int columnIndex) throws SQLException {
    String value = rs.getString(columnIndex);  //同上，这里提供的是columnIndex进行查询
    return new MyString(value);
}

@Override
public MyString getNullableResult(CallableStatement cs, int columnIndex) throws SQLException {
    String value = cs.getString(columnIndex);  //一般用于存储过程之类的操作，同上
    return new MyString(value);
}
```


## 在Mybatis配置文件中注册这个类型处理器
这样，我们自定义的类型处理器就编写好了，首先我们需要在Mybatis配置文件中注册这个类型处理器：

```xml
<typeHandlers>
    <typeHandler handler="com.test.handler.MyTypeHandler"/>
</typeHandlers>
```


## Mapper中配置编写好的类型处理器
接着我们需要在Mapper中配置这里编写好的类型处理器：

```java
@Results({
        @Result(column = "name", property = "name", typeHandler = MyTypeHandler.class)
})
@Select("select * from user where id = #{id}")
User selectUserById(int id);
```

```xml
<select id="selectUserByIdAndAge" resultMap="user">
    select * from user where id = #{id} and age = #{age}
</select>
<resultMap id="user" type="com.test.User">
    <result column="name" property="username" typeHandler="com.test.handler.MyTypeHandler"/>
</resultMap>
```
